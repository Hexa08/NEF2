# NEF2 Format Spec

The NEF2 binary format is designed for zero-copy deserialization. The layout guarantees that tensor data is correctly aligned for SIMD and GPU DMA operations.

## Header Schema

Every NEF2 file begins with a fixed-size header.

| Field | Type | Size | Description |
|---|---|---|---|
| `magic` | `char[4]` | 4 bytes | Always `NEF2` |
| `version` | `uint32_t` | 4 bytes | Format version (current: 2) |
| `tensor_count` | `uint64_t` | 8 bytes | Number of tensors in the file |
| `metadata_offset` | `uint64_t` | 8 bytes | Byte offset to the JSON metadata |

## Tensor Layout

Following the header is the tensor index, which contains the shape, dtype, and byte offset for each tensor payload.

<Callout type="warning" title="Memory Alignment">
  All tensor payloads in a NEF2 file are strictly padded to a 4096-byte boundary to ensure compatibility with NVMe Direct-to-GPU memory mapping.
</Callout>

## Example Parsing (C++)

```cpp language-cpp
#include <nef2/format.h>
#include <iostream>

int main() {
    nef2::FileReader reader("model.nef2");
    
    if (!reader.verify_magic()) {
        std::cerr << "Invalid NEF2 file." << std::endl;
        return 1;
    }
    
    std::cout << "Loaded model with " << reader.tensor_count() << " tensors." << std::endl;
    return 0;
}
```
