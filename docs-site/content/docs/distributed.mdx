# Distributed Systems

NEF2 provides first-class primitives for running massive models across distributed clusters.

## Sharding Strategies

You can distribute a model using Tensor Parallelism (TP) or Pipeline Parallelism (PP) with a single line of configuration.

```python language-python
import nef2
from nef2.distributed import Cluster

# Initialize a 4-GPU cluster
cluster = Cluster(world_size=4, strategy="tensor_parallel")

# Load model across the cluster
model = nef2.load("llama-70b.nef2", device=cluster)
```

## Multi-Node Setup

For multi-node execution, NEF2 utilizes an RDMA-optimized networking stack. Ensure the `nef2-daemon` is running on all worker nodes.

<Callout type="error" title="Firewall Configuration">
  The NEF2 distributed daemon requires TCP ports 29500-29510 to be open between all nodes in the cluster.
</Callout>

## RDMA Networking

If Mellanox ConnectX NICs are detected, NEF2 will automatically bypass the TCP stack and use InfiniBand/RoCE for node-to-node tensor transfers.
