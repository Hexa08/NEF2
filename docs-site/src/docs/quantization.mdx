## The Precision-Performance Tradeoff

Traditional deep learning uses 32-bit floating point (FP32) for training and 16-bit (FP16/BF16) for inference. While accurate, these formats are demanding on memory bandwidth and VRAM. TurboQuant allows you to push the limits of efficiency by using lower precision with minimal impact on model quality.

## Supported Formats

TurboQuant supports a wide range of precisions, optimized for modern hardware:

-   **FP8 (E4M3/E5M2)**: The new standard for high-performance training and inference on NVIDIA H100 and newer. It provides a near-perfect balance of range and precision.
-   **INT8**: Widely supported across almost all modern CPUs and GPUs. Excellent for deployment on edge devices.
-   **4-bit (NF4/GGUF)**: The gold standard for running massive LLMs on consumer hardware. TurboQuant utilizes Advanced Weight Quantization (AWQ) and AutoGPTQ techniques to maintain high accuracy at 4-bit.

## Dynamic Quantization

One of TurboQuant's unique features is **Dynamic Precision Scaling**. Instead of quantizing the entire model to a fixed format, TurboQuant can adjust precision on-the-fly based on the sensitivity of individual layers or even specific tokens (integrated with HyperCache).

```python
from nef2.memory import TurboQuant

# Quantize a tensor to FP8 for efficient storage in HyperCache
compressed_tensor = TurboQuant.quantize(original_tensor, target_precision="FP8")
```

## Hardware-Native Kernels

TurboQuant isn't just about storage; it's about execution. We provide high-performance kernels for:
-   **Mixed-Precision MatMul**: Multiplying INT8 weights by FP16 activations.
-   **Vectorized Dequantization**: Rapidly converting low-precision weights during the forward pass.
-   **Fused Quant-Op-Dequant**: Performing operations entirely in low precision to save bandwidth.

## Usage in Training (QAT)

TurboQuant supports **Quantization-Aware Training (QAT)**, allowing the model to learn to compensate for the quantization error during the training process. This results in significantly higher accuracy compared to post-training quantization (PTQ).

```python
import nef2.nn as nn

# Define a model with quantization-aware layers
model = nn.Sequential(
    nn.Linear(512, 1024, quantization="FP8"),
    nn.ReLU(),
    nn.Linear(1024, 10, quantization="INT8")
)
```

By mastering the art of low-precision computation, TurboQuant ensures that your models are not just powerful, but also exceptionally lean and fast.
