# Architecture Overview

NEF2 is designed around a three-tier architecture: the **Format Layer**, the **Runtime Engine**, and the **Hardware Backend**. This separation of concerns allows NEF2 to achieve extreme performance while maintaining framework independence.

## The Format Layer

The NEF2 format is a highly optimized, alignment-aware binary spec. It ensures that tensors are laid out exactly as the hardware expects them, eliminating runtime transposition overhead.

## The Runtime Engine

The core engine is responsible for memory virtualization (HyperCache) and kernel scheduling. It intercepts the execution graph and dynamically allocates resources across available devices.

## Hardware Backends

NEF2 abstracts the underlying silicon. A single NEF2 model can execute natively on NVIDIA, AMD, or Apple Silicon without modification.
