Unified Multi-Backend AI Runtime

NEF2 is an infrastructure-first AI framework designed to treat distributed hardware as a single, unified accelerator. Built on a polymorphic C++ Hardware Abstraction Layer, it delivers PyTorch-like flexibility with bare-metal hardware efficiency.

Core Philosophy

Scaling AI shouldn't mean wrestling with vendor lock-in. NEF2 abstracts the device, the stream, and the collective communication, allowing models to move seamlessly between NVIDIA, AMD, and Apple Silicon.

pip install nef2

Getting Started

Kickstart your journey with NEF2. We recommend using a virtual environment with Python 3.10+.

Installation

The standard package provides CPU execution and the Unified HAL headers.

pip install nef2

For GPU acceleration, see the Download Store for pre-compiled wheels matching your CUDA or ROCm version.

First Tensor

import nef2
x = nef2.randn((1024, 1024), device="cluster")
y = x * 2.0
print(y.numpy())

Unified Backend Runtime

Unlike traditional frameworks that add hardware support as a plugin, NEF2 is built around a centralized dispatch engine.

When you call an operation, the NEF2 engine identifies the optimal hardware backend (CUDA, HIP, Metal) and routes the execution through specialized vendor kernels while maintaining a single user-facing API.

Hardware Abstraction Layer (HAL)

The nef::Backend class is the heart of the system. It defines pure virtual interfaces for malloc, launch_kernel, and collectives.
