## The Unified Substrate

Modern AI infrastructure is often locked into specific hardware ecosystems. NEF2 breaks this lock-in by abstracting hardware-specific details behind a common set of primitives. The DAL allows developers to write code once and run it anywhere with near-native performance.

### DAL Primitives:
- **TensorOps**: Unified math operations (Add, Mul, MatMul, etc.).
- **MemoryOps**: Cross-device allocation, deallocation, and peer-to-peer transfers.
- **KernelLaunch**: A standard way to submit work to hardware command queues.
- **EventSystem**: Synchronization primitives for multi-stream and multi-device coordination.

## Backend Implementation

Each hardware vendor has a dedicated backend implementation within the `nef2/backend/` directory. These backends translate DAL calls into vendor-specific API calls (e.g., CUDA, HIP, Metal).

| Backend | Technology | Status |
| :--- | :--- | :--- |
| **CUDA** | NVIDIA Driver / PTX | Production Ready |
| **HIP** | AMD ROCm / hipBLAS | Production Ready |
| **Metal** | Apple Metal Performance Shaders | Production Ready |
| **SYCL** | Intel oneAPI / oneMKL | Active Development |
| **Vulkan** | SPIR-V / Cross-platform | Active Development |

## Memory Management

The DAL manages device memory through a hierarchical allocation system. It supports:

1.  **Pinned Host Memory**: Fast transfers between CPU and GPU.
2.  **Unified Virtual Addressing**: Simplifying data movement across different devices in the same node.
3.  **Direct P2P Transfers**: Moving data between GPUs via NVLink or Infinity Fabric without touching the CPU.

## Extensibility

Adding support for new hardware is a matter of implementing the DAL interface. Because NEF2 uses a modular architecture, new backends can be added as plugins without modifying the core runtime.

```python
# DAL usage example (internal)
import nef2.backend as backend

# Get the optimal backend for the current environment
dev_backend = backend.get_backend("cuda")

# Allocate memory via DAL
ptr = dev_backend.alloc(1024 * 1024) # 1MB

# Launch a generic kernel
dev_backend.launch("my_custom_kernel", inputs, outputs)
```

The DAL ensures that NEF2 remains future-proof, ready for whatever silicon comes next in the rapidly evolving AI landscape.
