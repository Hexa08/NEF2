## Beyond Python Loops

Traditional optimizers often suffer from significant overhead when updating millions of parameters from Python. NEF2 solves this by implementing optimizer update logic directly in C++ and CUDA kernels, ensuring that the "weight update" phase of training is as fast as the forward pass.

## Supported Optimizers

NEF2 provides optimized implementations for the most widely used algorithms:

### AdamW
The industry standard for training Transformers. NEF2's AdamW implementation includes integrated weight decay and specialized kernels to handle the second-moment estimations efficiently.

```python
import nef2

optimizer = nef2.AdamW(model.parameters(), lr=1e-4, weight_decay=0.01)
```

### SGD with Momentum
A classic choice for computer vision. Our SGD implementation supports Nesterov momentum and is highly optimized for multi-GPU scaling.

```python
optimizer = nef2.SGD(model.parameters(), lr=0.1, momentum=0.9)
```

## Hardware-Native Optimization

When running on NVIDIA or AMD hardware, NEF2 utilizes specialized kernels for the optimizer step. This reduces memory bandwidth pressure by combining the gradient application, weight decay, and momentum updates into a single fused operation.

## Training Workflow

The optimizer works seamlessly with the NEF2 Autograd engine. A typical training step looks like this:

```python
# 1. Forward pass
loss = model(inputs, targets)

# 2. Reset gradients
optimizer.zero_grad()

# 3. Backward pass (Autograd)
loss.backward()

# 4. Update weights (Hardware-accelerated)
optimizer.step()
```

## Custom Optimizers

The `Optimizer` base class allows you to implement custom optimization logic while still benefiting from NEF2's parameter management and hardware abstraction.

By focusing on the performance of the weight update cycle, NEF2 ensures that your training runs aren't just accurate, but also significantly faster than traditional frameworks.
