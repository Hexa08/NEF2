The AI Operating Substrate

NEF2 is not a framework. It is a Substrate. By bypassing the "heavy abstractions" of legacy AI libraries, NEF2 talks directly to the hardware via a unified C++/Rust core.

Hardware Agnostic

Run the same code on NVIDIA H100s, AMD MI300Xs, or Apple M3 Max without modification.

HAL Design

The Hardware Abstraction Layer (HAL) is the foundation. It defines the contract that all backends must follow.

// Example nef::Backend interface
class Backend {
    virtual void* malloc(size_t size) = 0;
    virtual void launch_kernel(Kernel* k, args...) = 0;
    virtual void all_reduce(Tensor* t) = 0;
};

By enforcing this interface, the high-level Graph Compiler never needs to know what brand of GPU it's running on.

Pipeline Parallelism (PP)

Sequential layer execution with micro-batching to eliminate "idle bubbles" in the GPU cluster.

Zero-Bubble Scheduler

NEF2's Rust orchestrator predicts future layer requirements and prefetches weights asynchronously.

Expert Parallelism (EP)

Specifically designed for Mixture of Experts (MoE) models.

Tokens are routed via high-speed all_to_all collectives to the specific GPU holding the expert weights for that token.

HyperCache KV

AI-Aware Scheduler

The Rust scheduler monitors GPU thermal states and memory bandwidth in real-time.

If GPU 0 becomes too hot, the scheduler dynamically re-routes Expert tokens to GPU 1 without interrupting the inference stream.
