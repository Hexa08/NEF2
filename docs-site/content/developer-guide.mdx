## Our Core Philosophy

-   **Performance is a Feature**: If it's not fast, it's a bug. Every line of code should be evaluated for its impact on the critical path.
-   **Hardware First**: We don't hide the hardware; we embrace it. Abstractions should enable hardware-specific optimizations, not prevent them.
-   **Minimalism**: Prefer small, focused modules over bloated, "all-in-one" solutions.

## Coding Standards

### Python
-   Use Python 3.10+ features (Type hints, Structural Pattern Matching).
-   Follow PEP 8, but prioritize clarity over rigid adherence.
-   All public APIs must have comprehensive docstrings.

### C++ (The Runtime Core)
-   C++17 standard.
-   Use RAII for all resource management (memory, handles, locks).
-   Avoid heavy template metaprogramming where a simple C-style interface would suffice for the ABI.

### Rust (The Safety & Networking Layer)
-   Idiomatic, safe Rust. `unsafe` is permitted but must be clearly justified and documented.
-   Leverage `tokio` for asynchronous tasks and `rayon` for data parallelism.

## Contribution Workflow

1.  **Research**: Before implementing a feature, research the state-of-the-art and the existing implementation.
2.  **RFC**: For major changes, open an Issue to discuss the architectural impact.
3.  **Implementation**: Write clean, tested code. Ensure your changes don't regress performance (use the `nef2.metrics.profile` tools).
4.  **Tests**: Every PR must include unit tests and, where appropriate, integration tests.
5.  **Documentation**: Update the relevant MDX pages in `docs-site/` to reflect your changes.

## Testing Strategy

NEF2 uses `pytest` for Python and a custom GTest-based suite for the C++ core.

```bash
# Run all tests
pytest tests/
# Run performance benchmarks
pytest tests/benchmarks/
```

## Pull Request Process

-   Ensure all CI checks pass (linting, type checking, tests).
-   Provide a clear description of **what** changed and **why**.
-   Include benchmark results if the PR affects the compute or memory path.

By following these standards, we ensure that NEF2 remains the most performant and reliable substrate for the future of AI.
