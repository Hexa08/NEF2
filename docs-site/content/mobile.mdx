## The Edge Challenge

Deploying state-of-the-art models on mobile devices is notoriously difficult due to strict thermal limits, limited VRAM, and diverse hardware architectures. NEF2 Mobile addresses these challenges by utilizing the same **DAL** and **HyperCache** technologies used in the datacenter, tailored for the mobile environment.

## Platform Backends

### iOS (CoreML & Metal)
NEF2 talks directly to Apple's Neural Engine (ANE) via CoreML and to the GPU via Metal. This ensures peak performance on iPhones and iPads. Our mobile runtime supports:
-   **MPSGraph Execution**: Leveraging Apple's latest graph compiler.
-   **Shared Memory**: Efficiently moving data between CPU and GPU on unified memory architectures.

### Android (Vulkan & NNAPI)
For the diverse Android ecosystem, NEF2 utilizes the **Vulkan Compute** backend and the Android Neural Networks API (NNAPI). This provides a portable yet high-performance path for acceleration across Qualcomm, Samsung, and Google silicon.

## Mobile-First Optimizations

The `nef2.mobile` export pipeline includes specialized optimizations for the edge:

1.  **Extreme Quantization**: Automatic conversion to 4-bit and 8-bit formats via TurboQuant.
2.  **Operator Pruning**: Removing operations not supported by mobile NPUs.
3.  **Static Memory Planning**: Pre-calculating all memory requirements to avoid dynamic allocations during inference, reducing latency jitter.

## Usage

Exporting a model for mobile is a single command:

```python
import nef2.mobile as mobile

# Export to a mobile-optimized format
mobile.export(model, "my_model.nef_mobile", target_platform="ios")
```

## The NEF Mobile Runtime

The mobile runtime is a lightweight, C++ implementation of the NEFCore, with a minimal binary footprint (under 5MB). It can be easily integrated into native iOS (Swift) or Android (Kotlin/Java) applications.

```swift
// Swift Example
let model = NEFModel(path: "my_model.nef_mobile")
let result = model.predict(inputTensor)
```
